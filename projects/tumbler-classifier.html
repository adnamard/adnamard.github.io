Reusable vs. Disposable Container Classifier

A thesis project asking whether a lightweight model can reliably tell a reusable tumbler apart from a disposable cup — accurately enough to support a real sustainability program.

Python TensorFlow/Keras MobileNetV2 HuggingFace
Live Demo GitHub Repository

The challenge

Business context

Sustainability initiatives that reward reusable-container use — smart bins, campus programs, café loyalty schemes — had no automated way to verify what people were actually using. Manual checks don't scale, so the program needed a model accurate and light enough to run in real time on modest hardware.

Questions I asked

🤖

Can a lightweight model classify reliably in real time?

🖼️

How much augmentation does a small dataset need?

🔍

Does it hold up on messy, real-world photos?

01 / DATA JOURNEY

Starting from a small, imbalanced dataset

The raw image set skewed heavily toward disposable cups, since they're simply more common to photograph. I checked class balance first, since an imbalanced training set quietly biases everything downstream.

💡 Disposable images outnumbered reusable ones nearly 2:1 — enough to bias the model toward predicting "disposable" by default without correction.

02 / FEATURE ENGINEERING & MODELING

Transfer learning with MobileNetV2

Rather than training from scratch on a small dataset, I froze the MobileNetV2 base pretrained on ImageNet and fine-tuned only the final layers, paired with augmentation (rotation, brightness jitter, flips) to correct the class imbalance and improve generalization.

💡 Validation accuracy tracked training accuracy closely after augmentation — a good sign the model wasn't just memorizing the training set.

Where the model still struggles

Confusion matrix on the test set

At 96% overall accuracy, most errors clustered in one direction — worth knowing before deploying this into a program that rewards people for correct classification.

Key takeaways

What the data actually said

🎯

96% accuracy on the held-out test set, achieved without training a network from scratch.

💡

Transparent or clear-colored containers were the most common misclassification, mostly confused for glassware.

☁️

Deploying to Hugging Face Spaces made the model testable by anyone without a local setup.

So what should change

Business recommendations

01

Expand the training set with more transparent and colored containers before rolling this out to a reward program — that's where errors concentrate.

02

Add a confidence threshold with human review for borderline predictions, rather than auto-approving every classification.

03

If moving toward on-device deployment (smart bins), quantize the model to keep inference fast on low-power hardware.

Reflection

This was my first real experience with transfer learning, and the biggest lesson was how much augmentation strategy matters when data is scarce — it moved the needle more than any architecture change. Next time I'd collect more edge-case photos (transparent containers, poor lighting) up front instead of patching them in after the first accuracy plateau.
