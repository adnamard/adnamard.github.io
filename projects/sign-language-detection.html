Tabir — Real-Time Sign Language Detection

Building a real-time SIBI (Indonesian Sign Language) recognition system from scratch — including the video dataset it had to learn from.

Python Keras TensorFlow MediaPipe
Live Demo GitHub Repository

The challenge

Business context

Deaf and hard-of-hearing individuals in Indonesia often face real friction in everyday interactions — not everyone knows SIBI, and interpreters aren't always available. No public dataset of SIBI gestures existed at the scale needed for a real-time classifier, so the project had to build one before it could train one.

Questions I asked

⚡

Can detection run in real time on consumer hardware?

🎥

How much custom video data does full coverage need?

🤚

Where does the model confuse similar handshapes?

01 / DATA JOURNEY

Building the dataset from scratch

I recorded and labeled custom video clips across common SIBI signs, then checked class coverage — static letter signs were easy to collect in volume, but motion-based word signs needed far more takes to capture enough variation.

💡 Motion-based signs had roughly half the sample count of static signs — a gap that shows up later as lower accuracy on exactly those classes.

02 / FEATURE ENGINEERING & MODELING

From video to landmarks to gesture

Rather than feeding raw video frames to the model, I used MediaPipe to extract hand landmark coordinates per frame, then trained a Keras classifier on those landmark sequences — a much smaller, faster-to-train feature space than raw pixels.

💡 Landmark-based features converged faster than a pixel-based approach would have — a good trade for a project needing real-time inference on modest hardware.

Where the model still struggles

Per-class accuracy

Real-time detection worked well overall, but accuracy wasn't even across every sign — worth knowing before positioning this as production-ready.

Key takeaways

What the data actually said

⚡

Real-time detection ran smoothly using MediaPipe's lightweight landmark extraction as the input feature.

🤚

Static signs with distinct handshapes were the most reliable; visually similar handshapes were the top source of errors.

🎥

Motion-based signs need noticeably more training data than static ones to reach the same accuracy.

So what should change

Recommendations

01

Prioritize collecting more motion-based sign clips before expanding vocabulary further — that's where accuracy currently lags.

02

Add a temporal sequence model (e.g. an LSTM over landmark frames) to better capture motion-based signs rather than treating each frame independently.

03

Validate the dataset and model outputs with SIBI-fluent community members before wider release, to catch regional or informal sign variations.

Reflection

Collecting the dataset took far longer than training the model — a reminder that for underrepresented languages, data collection is usually the real project. If I revisited this, I'd design the recording protocol with community input from day one, rather than after the first model felt "good enough."
